## 1. 核心概念

### 1.1 什么是 ClawPort？

ClawPort 是一个**多智能体协作控制台**，帮助你管理、监控和调度多个 AI 智能体。它连接到本地运行的 [OpenClaw](https://openclaw.ai) 网关，让多个智能体协同工作。

### 1.2 核心架构

```
┌─────────────────────────────────────────────────────────────┐
│                      ClawPort UI                            │
│  (组织架构图 | 对话 | 看板 | 定时任务 | 成本 | 记忆)          │
└─────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────┐
│                     OpenClaw Gateway                        │
│              (http://localhost:8000)                         │
└─────────────────────────────────────────────────────────────┘
                              │
          ┌───────────────────┼───────────────────┐
          ▼                   ▼                   ▼
    ┌──────────┐        ┌──────────┐        ┌──────────┐
    │ Agent 1  │        │ Agent 2  │        │ Agent 3  │
    │ (采集)   │        │ (分析)   │        │ (执行)   │
    └──────────┘        └──────────┘        └──────────┘
```

### 1.3 关键概念

| 概念 | 说明 |
|------|------|
| **Agent** | AI 智能体，有自己的角色、工具和任务 |
| **SOUL.md** | 智能体的人格定义文件，定义它的性格、行为方式 |
| **Cron Job** | 定时任务，让智能体按计划执行任务 |
| **Pipeline** | 流水线，多个任务按顺序执行的工作流 |
| **Kanban** | 看板，管理智能体的任务分配和进度 |

## 2. 快速开始

### 2.1 安装前提

```bash
# 1. 安装 Python (如果没有)
python --version

# 2. 安装 OpenClaw
pip install openclaw

# 3. 初始化 OpenClaw
openclaw init

# 4. 启动网关 (保持运行)
openclaw gateway run
```

### 2.2 启动 ClawPort

```bash
# 克隆中文版仓库
git clone https://github.com/kingwu791112/openclawport-cn.git
cd openclawport-cn

# 安装依赖
npm install

# 启动开发服务器
npm run dev
```

### 2.3 访问控制台

打开浏览器访问 http://localhost:3000

首次启动会显示引导向导，按照提示完成配置即可。

## 3. 创建你的第一个智能体

### 3.1 智能体配置文件

智能体定义在 `lib/agents.json` 文件中。每个智能体包含以下属性：

```json
{
  "id": "my-agent",
  "name": "小助手",
  "title": "个人助理",
  "reportsTo": null,
  "directReports": [],
  "soulPath": "agents/my-agent/SOUL.md",
  "color": "#f5c518",
  "emoji": "🤖",
  "tools": ["web_search", "web_fetch", "read", "write"],
  "description": "你的个人 AI 助理"
}
```

### 3.2 智能体属性说明

| 属性 | 说明 | 示例 |
|------|------|------|
| `id` | 唯一标识符 | `collector`, `trader` |
| `name` | 显示名称 | `信息采集员` |
| `title` | 职位/角色 | `首席数据分析师` |
| `reportsTo` | 上级智能体 ID | `team-leader` (为 null 表示顶级) |
| `directReports` | 下级智能体 ID 列表 | `["agent-a", "agent-b"]` |
| `soulPath` | SOUL.md 文件路径 | `agents/my-agent/SOUL.md` |
| `color` | 显示颜色 | `#f5c518` |
| `emoji` | 图标 | `📊` |
| `tools` | 可用工具列表 | `["web_search", "read", "write"]` |
| `description` | 描述 | `负责数据分析和报告` |

### 3.3 可用工具

| 工具 | 说明 |
|------|------|
| `web_search` | 搜索互联网 |
| `web_fetch` | 获取网页内容 |
| `read` | 读取文件 |
| `write` | 写入文件 |
| `exec` | 执行命令 |
| `message` | 发送消息 |
| `sessions_spawn` | 创建子会话 |
| `memory_search` | 搜索记忆 |
| `tts` | 文字转语音 |

### 3.4 创建 SOUL.md

在 `agents/` 目录下创建智能体的人格定义文件。例如 `agents/my-agent/SOUL.md`：

```markdown
# 我的智能体人格

## 基本信息
- 名字: 小助手
- 性格: 专业、耐心、细致

## 工作风格
- 思考方式: 逻辑清晰，循序渐进
- 沟通风格: 简洁明了，避免术语
- 决策原则: 数据驱动，保守估计

## 专业领域
- 擅长数据收集和分析
- 注重准确性和时效性
- 会主动提出优化建议

## 行为规范
- 不确定的问题会明确告知
- 会主动汇报工作进度
- 遇到问题及时求助上级
```

## 4. 多智能体协作团队

### 4.1 团队架构设计

多智能体协作的核心是**层级架构**：

```
                    ┌─────────────┐
                    │  总指挥者   │  (reportsTo: null)
                    └──────┬──────┘
                           │
        ┌──────────────────┼──────────────────┐
        ▼                  ▼                  ▼
   ┌─────────┐        ┌─────────┐        ┌─────────┐
   │ 团队A   │        │ 团队B   │        │ 团队C   │
   │ 采集组 │        │ 分析组 │        │ 执行组  │
   └────┬────┘        └────┬────┘        └────┬────┘
        │                   │                   │
        ▼                   ▼                   ▼
   ┌─────────┐        ┌─────────┐        ┌─────────┐
   │ 子智能体│        │ 子智能体│        │ 子智能体│

```

### 4.3 协作流程

1. **总指挥者** 接收任务或自动触发
2. **总指挥者** 分析任务，拆分成子任务
3. **分发任务** 给下级智能体
4. **子智能体** 执行任务并返回结果
5. **总指挥者** 汇总结果，进行最终处理

## 5. 定时任务与自动化

### 5.1 Cron 表达式

ClawPort 使用标准 Cron 表达式：

| 表达式 | 说明 | 示例 |
|--------|------|------|
| `* * * * *` | 每分钟 | 每分钟执行 |
| `0 * * * *` | 每小时 | 每小时整点执行 |
| `0 9 * * *` | 每天早上9点 | 每日简报 |
| `0 9 * * 1-5` | 工作日早上9点 | 工作日汇报 |
| `0 */4 * * *` | 每4小时 | 定期检查 |

### 5.2 创建定时任务

通过 OpenClaw CLI 创建定时任务：

```bash
# 创建定时任务
openclaw cron create \
  --agent my-agent \
  --name "每日简报" \
  --schedule "0 9 * * *" \
  --prompt "请汇总昨日重要信息，生成简报"

# 查看定时任务列表
openclaw cron list

# 手动触发定时任务
openclaw cron run "每日简报"
```

### 5.3 Pipeline 流水线

Pipeline 是多个任务按顺序执行的工作流：

```json
{
  "name": "每日资讯流程",
  "edges": [
    { "from": "采集员A", "to": "分析师", "artifact": "原始数据" },
    { "from": "分析师", "to": "总监", "artifact": "分析报告" }
  ]
}
```

### 5.4 任务投递 (Delivery)

定时任务完成后可以自动投递：

```bash
# 配置 Slack 投递
openclaw cron set-delivery \
  --name "每日简报" \
  --channel slack \
  --to "your-channel"

# 配置 Email 投递
openclaw cron set-delivery \
  --name "每日简报" \
  --channel email \
  --to "your@email.com"
```

## 7. 高级配置

### 7.1 环境变量

在项目根目录创建 `.env.local`：

```bash
# OpenClaw 网关地址
OPENCLAW_URL=http://localhost:8000

# OpenClaw 二进制路径
OPENCLAW_BIN=openclaw

# 自定义端口
PORT=3000
```

### 7.2 记忆功能

智能体可以记住之前的交互：

```json
{
  "id": "my-agent",
  "memoryPath": "memory/my-agent.md"
}
```

### 7.3 语音功能

启用语音输入输出：

```json
{
  "id": "my-agent",
  "voiceId": "onyx"  // 可选: alloy, echo, fable, onyx, nova, shimmer
}
```

### 7.4 看板任务分配

使用看板分配任务给智能体：

1. 打开 **看板** 页面
2. 创建任务卡片，指定负责的智能体
3. 设置优先级和截止时间
4. 智能体会自动执行任务

## 常见问题

### Q: 如何查看智能体的工作日志？

A: 在左侧菜单点击 **活动**，可以查看所有智能体的执行日志。

### Q: 如何手动触发定时任务？

A: 使用命令 `openclaw cron run <任务名>` 或在 ClawPort 界面点击运行按钮。

### Q: 智能体没有响应怎么办？

A: 1. 检查 OpenClaw 网关是否运行 2. 查看活动日志排查错误 3. 重启网关 `openclaw gateway run`

### Q: 如何备份智能体配置？

A: 定期备份 `lib/agents.json` 文件即可。
